## Base URL

```
https://api.village.ai/v2
```

## Authentication

All requests require a bearer token in the `Authorization` header:

```bash
Authorization: Bearer <token>
```

### Getting a Token

**If you're a partner integrating Village:**

1. Get your API key (starts with `sk_`) from Village Platform Admin
2. Generate a user token for each of your users:

```bash
curl https://api.village.ai/v2/auth/tokens \
  -H "secret-key: sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "external_user_id": "your_user_id",
    "email": "user@example.com"
  }'
```

3. Use the returned `access_token` in API requests

```bash
curl https://api.village.ai/v2/people/paths \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://linkedin.com/in/johndoe"
  }'
```

**If you're a direct user:**

Generate an API key (starts with `ak_`) from your Village dashboard and use it directly as the bearer token.

## Making Requests

```bash
curl https://api.village.ai/v2/people/paths \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://linkedin.com/in/johndoe"
  }'
```

### Identifiers

Most endpoints accept a `url` parameter that can be:

- LinkedIn profile URL: `https://linkedin.com/in/johndoe`
- LinkedIn company URL: `https://linkedin.com/company/acme`
- Company domain: `acme.com`

## Response Format

**Success (2xx):**

```json
{
  "data": { ... },
  "metadata": { ... },
}
```

**Error (4xx/5xx):**

```json
{
  "error": {
    "type": "validation_error",
    "status": 400,
    "detail": "Request validation failed",
    "trace_id": "trace_abc123"
  },
  "metadata": { ... },
}
```

Check the HTTP status code to determine success or failure.

## Rate Limits

- **Per User**: 100 requests/minute
- **Per Partner**: 10,000 requests/minute
- **Search Endpoints**: 20 requests/minute

Check response headers for current limits:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1704542430
```

## Errors

| Status | Type                   | Description                                    |
| ------ | ---------------------- | ---------------------------------------------- |
| 400    | `validation_error`     | Invalid request parameters                     |
| 401    | `authentication_error` | Invalid or missing token                       |
| 403    | `authorization_error`  | Insufficient permissions                       |
| 404    | `not_found_error`      | Resource not found                             |
| 429    | `rate_limit_error`     | Rate limit exceeded                            |
| 500    | `internal_error`       | Server error - contact support with `trace_id` |
